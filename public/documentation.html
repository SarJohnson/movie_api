Movie API

This is the server-side component of a “movies” web application. The web application will provide users with access to information about different horror movies. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

    Allows a user to be created

  • Endpoint URL: /users
  • Method type: POST
  • Query parameters: N/A
  • Format of request data: JSON
  • Format of response data: JSON

    Allows a user to update their username

  • Endpoint URL: /users/:id
  • Method type: PUT
  • Query parameters: :id
  • Format of request data: N/A
  • Format of response data: JSON

    Allows a user to favorite a movie

  • Endpoint URL: /users/:id/:movieTitle
  • Method type: POST
  • Query parameters: :id, :movieTitle
  • Format of request data: N/A
  • Format of response data: Text

    Allows a user to remove a favorited movie

  • Endpoint URL: /users/:id/:movieTitle
  • Method type: DELETE
  • Query parameters: :id, :movieTitle
  • Format of request data: N/A
  • Format of response data: Text

    Allows a user to be deleted

  • Endpoint URL: /users/:id
  • Method type: DELETE
  • Query parameters: :id
  • Format of request data: N/A
  • Format of response data: Text

    Shows all movies

  • Endpoint URL: /movies
  • Method type: GET
  • Query parameters: N/A
  • Format of request data: N/A
  • Format of response data: JSON

    Shows a specific movie

  • Endpoint URL: /movies/:title
  • Method type: GET
  • Query parameters: :title
  • Format of request data: N/A
  • Format of response data: JSON

    Shows a specific horror subgenre

  • Endpoint URL: /movies/subgenre/:subgenreName
  • Method type: GET
  • Query parameters: :subgenreName
  • Format of request data: N/A
  • Format of response data: JSON

    Shows a specific director

  • Endpoint URL: /movies/directors/:directorName
  • Method type: GET
  • Query parameters: :directorName
  • Format of request data: N/A
  • Format of response data: JSON
